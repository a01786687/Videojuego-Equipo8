**Historias de Usuario** 
----
Anura
--

**Historia de usuario #1 Juego con proposito**\
Yo como usuario, quiero un juego divertido, entretenido e inmersivo.

**Historia de usuario #2 Minimo de niveles**\
Yo como usuario quiero que haya multiples niveles, al menos dos.

**Historia de usuario #3 Dificultad progresiva**\
Yo como usuario quiero quiero que cada vez que juego sea una experiencia distinta, que los niveles vayan acoplándose de manera distinta.

**Historia de usuario #4 Acciones belicas**\
Yo como usuario quiero enfrentarme a enemigos y que esos enemigos reciban daño.

**Historia de usuario #5 HUD de daño y vida**\
Yo como usuario quiero tener un contador de daño, y un contador de vida. 

**Historia de usuario #6 Interactividad Cartas**\
Yo como usuario quiero que haya interacción entre cartas y el jugador.

**Historia de usuario #7 Dinámicas de Cartas**\
Yo como usuario quiero que esas cartas tengan ciertas dinamicas.

**Historia de usuario #8 Temporizador de Ejecución**\
Yo como usuario quiero tener un elemento más retador como un timer, un temporizador que me ayude a ejecutar mi código de manera más rapida. 

**Historia de usuario #9 Jefes de Nivel Diferenciados**\
Yo como usuario quiero que cada nivel tenga un jefe y que ese jefe tenga dinamicas distintas entre ellos.

**Historia de usuario #10 Mecánicas de Jefe Final**\
Yo como usuario quiero que haya un jefe final y que ese jefe final tenga también ciertas dinamicas.

**Historia de usuario #11 Menú de Inicio con Opciones**\
Yo como usuario quiero tener una pantalla de inicio que tenga opciones.

**Historia de usuario #12 Sistema de Pausa**\
Yo como usuario quiero tener una pantalla de pausa.

**Historia de usuario #13 Configuración en Pausa**\
Yo como usuario quiero que una pantalla de pausa tenga opciones cómo ajustar el sonido, guardar estado el juego, save stats y cosas así.

**Historia de usuario #14 Cinemáticas de Introducción y Cierre**\
Yo como usuario quiero tener una cinematica de entrada y una de salida.

**Historia de usuario #15 Autenticación Integrada en el Juego**\
Yo como administrador deseo que el metodo de autenticación esté dentro del juego.

**Historia de usuario #16 Persistencia de Catálogo de Cartas**\
Yo como administrador quiero salvar toda la informacion de las cartas.

**Historia de usuario #17 Estructura Web Multisección**\
Yo como usuario, quiero tener una página web que tenga multiples secciones.

**Historia de usuario #18 Navegación de Contenidos Web**\
Yo como usuario quiero que la página tenga una sección de inicio, una sección de personajes, una sección de historia, una sección de tutorial.

**Historia de usuario #18 Motor de Base de Datos MySQL**\
Yo como administrador de la base de datos quiero que trabajen en MySQL.

**Historia de usuario #19 Modelado Relacional e Indexación**\
Yo como administrador de la base de datos quiero tener multiples tablas con llaves primarias, llaves foraneas, llaves de indexación.

**Historia de usuario #20 Normalización de Datos (3NF)**\
Yo como administrador de la base de datos quiero que las tablas estén normalizadas hasta tercera forma normal.

**Historia de usuario #21 Implementación de Operaciones CRUD**\
Yo como administrador de la base de datos quiero que implementen todo el CRUD (Create, Read, Update, Delete).

**Historia de usuario #22 Conectividad API-Videojuego**\
Yo como administrador de la base de datos quiero que el CRUD esté conectado al videojuego con una API.

**Historia de usuario #23 Visualización de Estadísticas**\
Yo como administrador de la pagina quiero tener una sección extra que sea estadisticas.

**Historia de usuario #24 Business Intelligence e Interactividad**\
Yo como administrador de la página quiero que esa sección de estadisticas tenga visualizaciones interactivas que me ayuden a hacer una toma de decisiones ejecutiva.

**Historia de usuario #25 Integración Web del Juego**\
Yo como administrador de la página web quiero que el juego esté empotrado entre la página web.

**Historia de usuario #26 Gestión de Accesos (Login/Password)**\
Yo como administrador de la página web quiero tener algun metodo de autentificacion con login y password.

**Historia de usuario #27 Despliegue Local y Cloud**\
Yo como administrador de la página web quiero poder desplegar esa página web tanto de manera local como en la nube.

**Historia de Usuario 28 Iniciar y terminar partida**\
Yo como jugador quiero poder iniciar (start game) y terminar una partida (game over).

**Historia de Usuario 29 Progresión tipo roguelite**\
Yo como jugador quiero volverme más fuerte cada vez que muera para progresar.

**Historia de Usuario 30 Pérdida y guardado de progreso**\
Yo como jugador quiero que al morir parte de mi progreso se pierda y otra parte se guarde.

**Historia de Usuario 31 Sistema de cartas que afecte el gameplay**\
Yo como jugador quiero usar cartas que puedan afectar a mi personaje o a los enemigos para modificar las acciones durante el combate y afectar el gameplay.

**Historia de Usuario 32 Obtención de cartas por nivel**\
Yo como jugador quiero recibir nuevas cartas cada vez que complete un nivel.

**Historia de Usuario 33 Recoleccion de estadisticas para mejoras y balanceo**\
Yo como desarrollador quiero que el juego recabe estadísticas de las partidas para analizar la dificultad del juego y poder balancear enemigos y mecánicas.

**Historia de Usuario 34 Estadísticas específicas por nivel y jefes**\
Yo como dearrollador quiero tener estadísticas específicas de niveles, enemigos y jefes para identificar si alguno es demasiado difícil o demasiado fácil y ajustar su balance

**Historia de Usuario 35 Sprites finales del juego**\
Yo como jugador quiero que el juego utilice los sprites finales de los personajes, enemigos y elementos para tener una experiencia agradable y consistente con el diseño del juego.

**Historia de Usuario 36 Estética de la página web**\
Yo como usuario quiero que la página web tenga una estética coherente con el juego para mantener una identidad visual clara y una experiencia agradable

**Historia de Usuario 37 Estilo pixel art/retro en el frontend**\
Yo como usuario quiero que el frontend mantenga una estética pixel art consistente con el juego para mantener la identidad visual y mejorar la experiencia del jugador.
