**Historias de Usuario**  

---- 

Anura 

-- 

 

**Historia de usuario #1 Juego con proposito**\ 

Yo como usuario quiero un crear un videojuego para que sea divertido, entretenido e inmersivo. 

 

**Historia de usuario #2 Minimo de niveles**\ 

Yo como usuario quiero que haya multiples niveles, al menos dos, para que el juego tenga etapas de evolución. 

 

**Historia de usuario #3 Dificultad progresiva**\ 

Yo como usuario quiero que tener niveles con dificultad progresiva para que cada vez que juego sea una experiencia distinta. 

  

**Historia de usuario #4 Enfrentamiento con enemigos**\ 

Yo como usuario quiero enfrentarme a enemigos para transmitir la sensación de victoria contra un adversario. 

 

**Historia de usuario #5 HUD de daño**\ 

Yo como usuario quiero tener un contador de daño para monitorear mi avance y tomar decisiones contra los enemigos. 

 

**Historia de usuario #6 Interactividad Cartas**\ 

Yo como usuario quiero que haya interacción entre cartas y el jugador para obtener o potenciar las habilidades de mi personaje. 

 

**Historia de usuario #7 Dinámicas de Cartas**\ 

Yo como usuario quiero que esas cartas tengan ciertas dinámicas para que sean flexibles y puedan apoyar a mi personaje o afectar a los enemigos. 

 

**Historia de usuario #8 Temporizador de Ejecución**\ 

Yo como usuario quiero tener un timer que funcione como elemento retador para que pueda comparar una corrida con otra y determinar mi mejoría. 

 

**Historia de usuario #9 Jefes en cada nivel**\ 

Yo como usuario quiero que cada nivel tenga un jefe para darle un cierre retador a cada nivel con la temática respectiva del nivel. 

 

**Historia de usuario #10 Jefe Final**\ 

Yo como jugador quiero enfrentarme a un jefe final que concluya el juego de manera emocionante y desafiante. 

 

**Historia de usuario #11 Menú de Inicio con Opciones**\ 

Yo como usuario quiero tener una pantalla de inicio que tenga opciones para poder comenzar o continuar con una corrida y/o para ajustar el brillo. 

 

**Historia de usuario #12 Sistema de Pausa**\ 

Yo como usuario quiero tener una pantalla de pausa para poder continuar con mi corrida en caso de que me tenga que ausentar temporalmente. 

 

**Historia de usuario #13 Configuración en Pausa**\ 

Yo como usuario quiero que una pantalla de pausa tenga opciones para ajustar el sonido, guardar estado el juego, checar mis cartas activas. 

 

**Historia de usuario #14 Cinemáticas de Introducción y Cierre**\ 

Yo como desarrollador quiero tener una cinematica de entrada y una de salida en el juego para darle una historia al juego. 

 

**Historia de usuario #15 Persistencia de Catálogo de Cartas**\ 

Yo como administrador quiero salvar toda la informacion de las cartas para determinar como impactan en el jugador, corrida o enemigos. 

 

**Historia de usuario #16 Estructura Web Multisección**\ 

Yo como usuario quiero tener una página web que tenga multiples secciones para consultar mecánicas y tutorial, historia y personajes. 

 

**Historia de usuario #17 Motor de Base de Datos MySQL**\ 

Yo como administrador de la base de datos quiero que trabajen en MySQL. 

 

**Historia de usuario #18 Modelado Relacional e Indexación**\ 

Yo como administrador de la base de datos quiero tener multiples tablas con llaves primarias, llaves foráneas, llaves de indexación para poder almacenar datos eficientemente y poder hacer consultas de información. 

 

**Historia de usuario #19 Normalización de Datos (3NF)**\ 

Yo como administrador de la base de datos quiero que las tablas estén normalizadas hasta tercera forma normal para almacenar los datos del juego y usuario eficientemente. 

 

**Historia de usuario #20 Implementación de Operaciones CRUD**\ 

Yo como administrador de la base de datos quiero que implementen todo el CRUD (Create, Read, Update, Delete). 

 

**Historia de usuario #21 Conectividad API-Videojuego**\ 

Yo como administrador de la base de datos quiero que el CRUD esté conectado al videojuego con una API para poder almacenar los datos de cada usuario y consultar estadísticas. 

 

**Historia de usuario #22 Visualización de Estadísticas**\ 

Yo como administrador de la pagina quiero tener una sección extra que sea estadisticas para que los usuarios puedan consultar información de sus corridas, cartas y personaje. 

 

**Historia de usuario #23 Business Intelligence e Interactividad**\ 

Yo como administrador de la página quiero que esa sección de estadisticas tenga visualizaciones interactivas para que me ayuden a hacer una toma de decisiones ejecutiva. 

 

**Historia de usuario #24 Integración Web del Juego**\ 

Yo como administrador de la página web quiero que el juego esté empotrado entre la página web. 

 

**Historia de usuario #25 Gestión de Accesos (Login/Password)**\ 

Yo como administrador de la página web quiero tener algún método de autentificación con login y password para que el jugador pueda registrarse fácilmente y guardar su progreso. 

 

**Historia de usuario #26 Despliegue Local y Cloud**\ 

Yo como administrador de la página web quiero poder desplegar esa página web tanto de manera local como en la nube. 

 

**Historia de Usuario #27 Iniciar y terminar partida**\ 

Yo como jugador quiero poder iniciar (start game) y terminar una partida (game over) para que marque un ciclo dentro del juego. 

 

**Historia de Usuario #28 Progresión tipo roguelite**\ 

Yo como jugador quiero volverme más fuerte cada vez que muera para progresar. 

 

**Historia de Usuario #29 Pérdida y guardado de progreso**\ 

Yo como jugador quiero que al morir parte de mi progreso se pierda y otra parte se guarde para que el progreso sea desafiante. 

 

**Historia de Usuario #30 Sistema de cartas que afecte el gameplay**\ 

Yo como jugador quiero usar cartas que puedan afectar a mi personaje o a los enemigos para modificar las acciones durante el combate y afectar el gameplay. 

 

**Historia de Usuario #31 Obtención de cartas por nivel**\ 

Yo como jugador quiero recibir nuevas cartas cada vez que complete un nivel para que mi personaje sea más fuerte en los niveles más complicados. 

 

**Historia de Usuario #32 Recoleccion de estadisticas para mejoras y balanceo**\ 

Yo como desarrollador quiero que el juego recopile estadísticas de las partidas para analizar la dificultad del juego y poder balancear enemigos y mecánicas. 

 

**Historia de Usuario #33 Estadísticas específicas por nivel y jefes**\ 

Yo como dearrollador quiero tener estadísticas específicas de niveles, enemigos y jefes para identificar. si alguno es demasiado difícil o demasiado fácil y ajustar su balance 

 

**Historia de Usuario #34 Sprites finales del juego**\ 

Yo como jugador quiero que el juego utilice los sprites finales de los personajes, enemigos y elementos para tener una experiencia agradable y consistente con el diseño del juego. 

 

**Historia de Usuario #35 Estética de la página web**\ 

Yo como usuario quiero que la página web tenga una estética coherente con el juego para mantener una identidad visual clara y una experiencia agradable. 

 

**Historia de Usuario #36 Estilo pixel art/retro en el frontend**\ 

Yo como usuario quiero que el frontend mantenga una estética pixel art consistente con el juego para mantener la identidad visual y mejorar la experiencia del jugador. 

 

**Historias de Usuario #37 Niveles aleatorios**\ 

Yo como usuario quiero que los niveles vayan acoplándose de manera distinta para que el juego no sea repetitivo. 

 

**Historia de usuario #38 Enfrentamiento con enemigos**\ 

Yo como usuario quiero que esos enemigos reciban daño para darle temática de combate al juego. 

 

**Historia de usuario #39 HUD de vida**\ 

Yo como usuario quiero un contador de vida para administrar mi personaje y adaptar mi estrategia en el run. 

 

**Historia de usuario #40 Jefes diferentes**\ 

Yo como usuario quiero que cada jefe tenga dinámicas distintas entre ellos para que la experiencia de combate no te acostumbre al siguiente jefe. 

 

**Historia de usuario #41 Mecánicas de Jefe Final**\ 

Yo como jugador quiero que el jefe final también tenga ciertas dinámicas.
